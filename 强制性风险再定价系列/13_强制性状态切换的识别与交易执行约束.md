# 第十三篇：强制性状态切换的识别与交易执行约束

## 摘要

强制性风险再定价框架最终要落到执行，但执行不是把概念直接翻译成买卖动作，而是建立一套证据门槛、失效条件和复盘机制。本文把前文的制度推导、成本锚、CARCB、期权路径、参与者约束、波动率状态和交易流验证，压缩为可操作的交易执行流程：识别成本锚，确认风险约束带，观察被动风险调整流，判断吸收或接受，定义失效条件，控制执行成本，盘后迭代。本文讨论交易执行验证清单、入场类型、止损逻辑、仓位降级条件和复盘字段。它不是频繁交易系统，而是过滤系统；不是为了预测所有行情，而是为了只处理那些风险约束已经迫使市场表态的变化。

## 一、问题提出

许多交易框架失败，不是因为概念完全错误，而是因为概念到执行之间缺少硬约束。交易员知道要看成本锚，却在没有明确成本锚时入场；知道要看流动性验证，却在突破瞬间追单；知道要设置止损，却把止损放在自己能忍受的位置，而不是交易假设失效的位置。

强制性状态切换的识别，要求交易员把“我觉得可能”降级，把“市场已经给出证据”升级。价格变化只有在激活风险约束、产生被动风险调整流，并被成交与流动性状态验证之后，才进入执行讨论。

本文要解决的问题是：如何把这套框架变成交易前、交易中、交易后的流程？如何定义入场、止损和降级？如何让盘后复盘真正修正模型，而不是变成情绪记录？

## 二、执行前提：五个证据门槛

第一，成本锚必须明确。交易员需要知道当前使用的是 VWAP、POC、结算价、突破前平衡区还是事件后成本区。若多个成本锚冲突，必须先确定交易周期和主锚。

第二，CARCB 必须可描述。价格偏离要用波动单位、保证金损益单位、期权预期波动区间或历史区间宽度表达。不能只说“走远了”。

第三，被动风险调整流必须出现。观察大额主动成交、主动成交净差、止损扫单、交易商动态对冲、强制平仓迹象、流动性撤出或成交推进效率变化。

第四，市场态度必须分辨。被动流是被吸收，还是被接受？价格是收复，还是在新区域形成成交？订单簿是补充深度，还是撤出深度？

第五，失效条件必须提前定义。若交易的是回归，什么状态说明偏离重新被接受？若交易的是延续，什么状态说明新区域被拒绝？没有失效条件，不进入执行。

| 执行门槛 | 必答问题 | 合格答案示例 |
|---|---|---|
| 成本锚 | 从哪里测量风险？ | 当日 VWAP 与压缩区 POC 重合 |
| CARCB | 偏离是否进入约束区？ | 偏离 1.8 倍日内波动，保证金损益比例上升 |
| 被动流 | 谁被迫调整？ | 止损扫单 + 主动成交净差倾斜 |
| 验证 | 市场吸收还是接受？ | 低位吸收并收复旧边界 |
| 失效 | 什么情况说明错？ | 价格重新接受低位并形成新成本区 |

执行章把全系列证据压缩成三层权限。位置证据给观察区，状态证据给路径权重，执行证据给入场权限。没有位置证据，不启动入场判断；只有位置证据，没有状态证据，只能观察；有位置和状态，但没有执行证据，仍然不能入场；执行证据出现后，还要通过失效条件和执行成本过滤。

| 权限层 | 变量 | 可以做什么 | 不能做什么 |
|---|---|---|---|
| 位置证据 | 成本锚、CARCB、VWAP、POC、Settlement、期权节点、区间边界 | 启动观察 | 不能入场 |
| 状态证据 | Gamma、Skew、波动率、时间结构、收盘强度、跨市场确认、OI | 调整路径权重 | 不能替代盘口 |
| 执行证据 | 主动成交、CVD、推进效率、LOB、吸收、接受、二次测试、收复 | 给入场权限 | 不能脱离失效条件 |

## 三、入场类型与路径匹配

**回归型入场。** 条件是价格偏离成本锚，进入 CARCB，被动风险调整流出现，但新区域被拒绝。入场不能在极端位置凭感觉发生，而应在价格收复关键边界、主动成交推进效率下降、限价流动性吸收明确之后。目标不是预测完全回到成本锚，而是研究向成本锚回归的风险收益。

**延续型入场。** 条件是价格偏离被接受，主动成交推进效率高，旧边界回测有效，新区域成交形成。延续型入场的风险是追在流动性最薄处，因此更适合等待回测、二次压缩或新成本锚形成后的确认。

**衰竭型入场。** 条件是深层约束区中最后一批强制成交失效，累积主动成交净差与价格背离，价格收复最近加速边界。衰竭型入场必须等待确认，因为深层约束区可以继续扩展。

**中继型入场。** 条件是扩张后新区域形成二次压缩，新成本锚稳定，旧边界防守有效。中继型入场比第一段突破更克制，依赖新成本带而不是原始冲动。

| 入场类型 | 前提条件 | 主要证据 | 失效条件 |
|---|---|---|---|
| 回归型 | 偏离被拒绝 | 吸收 + 收复 | 新区域重新接受 |
| 延续型 | 偏离被接受 | 推进效率 + 回测有效 | 旧边界被收复 |
| 衰竭型 | 深层强制流失效 | 净差背离 + 收复 | 极端区继续接受 |
| 中继型 | 新成本带形成 | 二次压缩 + 新锚稳定 | 跌回旧成本区 |

## 四、止损、仓位与执行成本

止损应放在表态失败点外侧。回归型交易的止损，是市场重新接受偏离；延续型交易的止损，是市场拒绝偏离并收复旧边界；衰竭型交易的止损，是最后一批强制流重新变得有效；中继型交易的止损，是新成本锚失守。

仓位应随证据质量分层。若只有成本锚和 CARCB，没有交易流验证，仓位为零；若出现被动流但吸收/接受未分辨，只能观察；若验证成立但订单簿深度薄、价差大，应降低仓位；若路径、流动性和失效条件都清楚，才允许标准仓位。

执行成本必须进入计划。强制性状态切换经常发生在价差扩大、深度变薄、滑点上升的时段。理论风险收益如果没有扣除滑点，容易高估。交易员应记录入场时价差、预期滑点、可成交深度和撤单条件。没有这些，微观结构框架在实盘中会变得过度乐观。

## 五、盘面序列案例

**示例性路径一：回归型执行。** 价格跌破 3000 成本锚进入下方 CARCB，主动卖出放大。交易员不在第一轮低点买入，而是等待低位卖出被吸收、价格收复 2990、回测不破、主动卖出推进效率下降。入场逻辑是偏离失败，止损放在价格重新跌回 2980 并形成新成交接受之后。

**示例性路径二：延续型执行。** 价格突破 4200 压缩区上沿，主动买入推进效率高，回撤不破 4200，4215 上方形成新成交成本区。交易员不追第一根突破，而是在 4208 到 4212 回测中观察旧边界防守。若成立，入场研究下一风险约束层；若价格跌回 4200 下方，延续假设失效。

**示例性路径三：降级执行。** 价格进入深层 CARCB，交易流支持延续，但订单簿深度极薄，价差扩大，滑点不可控。即使方向判断较强，也应降低仓位或放弃追价，等待二次确认。交易机会不只由方向决定，也由可执行性决定。

## 六、盘后复盘模板

复盘应拆分错误类型，而不是只记录盈亏。

| 字段 | 记录内容 |
|---|---|
| 主成本锚 | VWAP / POC / 结算价 / 事件后成本区 |
| 偏离单位 | ATR、实现波动率、保证金损益比例 |
| CARCB 层级 | 浅层、中层、深层、完全释放 |
| 参与者假设 | 零售、Prop、CTA、Hedge Fund、Dealer、HFT |
| 交易流证据 | 主动成交净差、推进效率、大额事件 |
| LOB 证据 | 深度、撤单、吸收、价差 |
| 路径分类 | 回归、延续、衰竭、中继、流动性空洞 |
| 入场原因 | 哪个证据触发执行 |
| 失效条件 | 事前定义与实际触发 |
| 错误类型 | 锚点错误、层级错误、验证错误、执行成本错误 |

复盘的目的不是证明交易有道理，而是校准下一次阈值。若多次亏损来自同一类错误，例如把浅层 CARCB 当深层，说明计算层需要修正；若多次亏损来自滑点，说明执行层需要降级；若多次亏损来自假接受，说明 LOB 验证标准不够严格。


## 七、日内工作流：从盘前准备到盘后迭代

一个可执行的日内流程可以分为三段。

盘前阶段，交易员应标记主成本锚、次级成本锚、前一结算价、主要期权节点、预期波动区间、重要数据发布时间和可能的 CARCB。盘前不需要预测方向，但必须知道如果价格向上或向下偏离，哪些参与者可能被触发，哪些位置需要重点观察。

盘中阶段，交易员只做两类事：更新状态和等待验证。价格在正常波动区内，不因噪音入场；价格进入浅层 CARCB，记录主动成交和深度变化；进入中层或深层 CARCB，重点观察被动风险调整流是否出现；出现被动流后，等待吸收或接受分辨。若市场没有分辨，就保持观察。

盘后阶段，交易员必须把盘中判断写回复盘。若当天没有交易，也要记录为什么没有交易：没有成本锚、没有进入 CARCB、没有被动流、没有验证，还是执行成本过高。这个记录不是为了补充叙述，而是为了确认当日是否遵守了执行门禁。

三阶段闭环可以写成固定清单。盘前定义关键成本锚、CARCB 层级、期权节点、事件窗口、高周期状态、主路径、备选路径和失效路径。盘中只在关键区域启动判断，观察交易流与 LOB，判断吸收或接受，执行或放弃，入场后要求快速验证，不符合预期则降级。盘后只回答三件事：行情是否符合原计划，原计划何处失效，自己是否违反执行门禁。这个闭环的作用，是让交易员在市场继续波动时仍然知道自己何时应该停止。

## 八、执行决策树

可以用一棵简化决策树约束盘中行动：

1. 当前价格是否围绕明确成本锚运行？若否，等待。
2. 价格是否偏离成本锚并进入可描述的 CARCB？若否，等待。
3. 是否出现被动风险调整流？若否，只记录。
4. 被动流是否被吸收？若是，进入回归型计划。
5. 被动流是否被接受？若是，进入延续型计划。
6. 吸收和接受是否矛盾？若矛盾，降级或放弃。
7. 执行成本是否可控？若不可控，降仓或等待二次确认。
8. 失效条件是否明确？若不明确，不交易。

这棵树的价值，是减少盘中叙事自由度。市场越剧烈，交易员越容易临时修改理由。决策树要求每一步都有证据，没有证据就不能跳到下一步。

## 九、风险预算与仓位分层

执行约束还必须进入仓位层。可以把仓位分为四级。零级是观察仓位，即不交易。一级是试探仓位，适用于成本锚、CARCB 和被动流成立，但吸收/接受刚刚出现且执行成本较高。二级是标准仓位，适用于路径证据完整、失效条件清晰、流动性可接受。三级并不建议常态使用，只适用于多周期证据共振、流动性充足且风险预算允许的情况。

仓位分层的目的不是放大信心，而是让证据质量与风险暴露一致。若交易员在证据不完整时使用标准仓位，本质上是让主观判断替代框架；若证据完整但执行成本过高，也应降低仓位。强制性风险再定价框架强调等待，但等待之后仍然需要风险预算管理。

## 十、正式执行约束

执行约束应写成可以被盘中检查和盘后追责的规则。

| 规则 | 正式约束 | 违反后的处理 |
|---|---|---|
| 1 | 不在中间价预测；价格未接近成本锚、CARCB、区间边界或期权节点，不启动入场判断 | 记录为非关键区域交易 |
| 2 | 不在非关键区域交易；没有预设关键位，不允许临时解释价格波动 | 取消当次计划 |
| 3 | 只有位置，没有交易流与 LOB 验证，不入场 | 维持观察 |
| 4 | 只有观点，没有失效位，不入场 | 重写假设 |
| 5 | 入场后不能快速脱离成本，降级为平本、小亏或小盈处理 | 不扩大仓位 |
| 6 | 连续两次同类失败，停止交易并重写市场状态 | 当日该路径暂停 |
| 7 | 达到计划观察时间或交易次数后，不因市场继续波动而继续交易 | 进入盘后复盘 |
| 8 | 高波动、价差扩大、深度下降环境自动降仓 | 标准仓降为试探或放弃 |
| 9 | 低波转高波时，禁止亏损加仓和幻想回归 | 以失效条件退出 |
| 10 | 流动性空洞中禁止追价，除非深度恢复后新区域仍被接受 | 等待二次验证 |
| 11 | CARCB 层级必须在触及前定义，不能价格到达后临时改层级 | 该层级判断作废 |
| 12 | 主路径、备选路径、路径转换条件和失效条件必须在盘前或触发前写下 | 不允许盘后补写 |
| 13 | 盘后不能用新路径为旧判断辩护，只能记录原路径是否成立 | 归为事后改叙事 |
| 14 | 如果路径只能事后解释，不能盘前定义触发和失效，不得作为执行路径 | 从 checklist 删除 |
| 15 | 数据不足时，不降低入场标准，只降低交易级别 | 仓位降级或放弃 |

这些规则的目标不是避免亏损，而是控制错误暴露。方向判断正确但在非关键区域追入、止损过远或滑点不可控，仍然属于执行错误。执行章评估的是“是否按证据链交易”，不是“事后价格是否证明自己有道理”。


## 十一、三类失败交易的处理

第一类是识别失败。交易员把普通变化误判为强制变化，例如没有进入 CARCB 就入场，或没有被动风险调整流就交易。处理方式是提高前置门槛，减少解释自由度。

第二类是验证失败。交易员识别到了风险约束，但没有等市场分辨吸收或接受。例如看到深层偏离就提前逆向，或看到突破就直接追随。处理方式是把成交推进效率、收复和新区域成交写成硬条件。

第三类是执行失败。方向判断并不差，但入场太急、滑点太大、仓位太重、止损不在失效点。处理方式不是修改市场观点，而是修改成交策略和仓位规则。

复盘时把亏损归入这三类，比简单说“看错方向”更有价值。很多交易并不是方向完全错，而是证据不足、周期错配或执行成本过高。只有错误被拆分，框架才会进化。

## 十二、交易前一句话假设

每次执行前，交易员应能用一句话写出假设：价格从哪个成本锚偏离，进入哪一层 CARCB，哪类参与者可能被迫调整，盘面显示这批调整被吸收还是接受，失效条件是什么。若这句话写不出来，说明交易还停留在冲动或模糊判断。

例如：“价格从 4200 压缩区 POC 向上偏离，进入上行中层 CARCB，空头回补与负 Gamma 对冲可能形成被动买入，突破后新区域成交接受，若价格收复 4210 下方则延续假设失效。”这样的假设可执行、可验证、可复盘。模糊的“感觉要突破”则不可执行。

这一句话不是形式主义，而是防止交易员盘中不断改理由。一个清楚的假设，允许市场证明它对，也允许市场证明它错。

## 十三、执行后的三问

交易结束后只问三个问题。第一，入场前证据是否完整？若不完整，问题在识别层。第二，交易过程中是否尊重失效条件？若没有，问题在纪律层。第三，真实成交是否符合预期滑点和深度？若不符合，问题在执行层。

这三个问题比盈亏更重要。盈利交易也可能证据不足，亏损交易也可能执行正确。长期看，框架质量取决于证据流程是否稳定，而不是单笔结果是否漂亮。若交易员能把每笔交易归入识别、纪律或执行问题，系统就有迭代方向。

## 十四、从少交易到高质量交易

强制性风险再定价框架天然会降低交易频率。很多行情没有成本锚，很多偏离没有被动流，很多被动流没有验证，很多验证发生时执行成本过高。过滤掉这些情况，不是错过，而是把风险暴露留给更高质量的状态切换。

少交易不是目标本身，高质量交易才是目标。少交易只是高门槛的自然结果。交易员若不能接受空白时间，就会把框架重新变成寻找理由的工具。

执行框架最终要形成个人化参数。不同交易员的数据条件、持仓周期、滑点成本和心理承受能力不同。同一套强制性风险再定价逻辑，在短线交易员手中可能重视 LOB，在波段交易员手中可能重视结算价和新成本锚。个性化不是随意修改规则，而是在相同证据链下调整执行尺度。

最终，执行约束的价值体现在两件事：减少不该做的交易，保留真正有证据的交易。许多交易系统只强调如何进场，却不强调如何不进场。本框架的执行层恰恰相反，先定义不能交易的状态，再定义可以交易的状态。这个顺序，是风险控制的一部分。

当交易员能够清楚说出“今天为什么没有交易”，执行框架才算真正建立。空仓不是缺乏判断，而是证据门槛没有被满足。

这也是执行约束的最终含义：它不奖励频繁行动，而奖励清楚、可验证、可退出的行动。没有这些条件，等待就是正确动作。

执行系统真正成熟时，交易员不会因为没有仓位而焦虑，也不会因为价格剧烈变化就临时降低门槛。门槛稳定，盈亏才有可分析的结构；这套框架最终服务于执行边界，而不是为盘中冲动提供解释空间。

## 十五、盘前计划模板

执行约束必须从盘前开始。盘前计划不是预测涨跌，而是写清楚今天哪些区域值得观察，哪些条件出现才允许交易，哪些条件出现必须降级或放弃。一个合格的盘前计划至少包含成本锚、CARCB、期权节点、事件风险、流动性条件和路径假设。

| 模块 | 需要写清的问题 | 示例字段 |
|---|---|---|
| 成本锚 | 今日主锚和次锚在哪里 | 前结算价、日内 VWAP、三日 POC、事件后成本区 |
| CARCB | 上下方约束层如何划分 | 浅层、中层、深层、完全风险释放区 |
| 期权环境 | 哪些节点可能改变路径弹性 | Call Wall、Put Wall、Gamma Flip、Skew、0DTE |
| 事件风险 | 哪些时间点可能重置成本锚 | 数据发布时间、库存报告、议息、拍卖 |
| 流动性条件 | 哪些时段执行成本较高 | 开盘、数据后、午盘、收盘前 |
| 路径假设 | 今天最多跟踪哪 2-3 条路径 | 偏离失败、延续、中继、Gamma 挤压 |

盘前计划还要写明“不交易条件”。例如：没有清晰成本锚不交易；CARCB 只到浅层且无被动流不交易；期权节点没有期货成交验证不交易；价差扩大且深度不足时不追价；路径失效后不立即反向。这些限制比入场理由更重要，因为它们决定框架是否能过滤冲动。

## 十六、盘中 6 问与入场 checklist

盘中执行可以压缩为六个问题。第一，价格从哪个成本锚偏离？第二，偏离进入哪一层 CARCB？第三，哪类参与者可能被迫调整？第四，是否出现被动风险调整流？第五，这批流被市场吸收还是接受？第六，若判断错误，什么盘面状态会证明错误？

| 入场前检查 | 必须满足 | 不满足时处理 |
|---|---|---|
| 成本锚明确 | 主锚可写清，周期匹配 | 不交易 |
| 约束层明确 | 偏离可用波动/保证金/微观结构表达 | 只观察 |
| 路径假设明确 | 回归、延续、衰竭、中继等只能选其一为主 | 不混合叙事 |
| 交易流验证 | 主动流、推进效率、吸收或接受可描述 | 等待下一窗口 |
| LOB / 执行可行 | 深度、价差、滑点可承受 | 降级或放弃 |
| 失效条件 | 价格或流动性状态能证明假设错误 | 不入场 |

示例性路径四：盘前计划写明 4200 为主成本锚，上方 4230 为中层 CARCB，4250 附近有 Call Wall。盘中价格突破 4230 后，主动买入放大但 4245 上方被吸收，价格跌回 4230，下次回测无法站回。此时若执行回归型交易，入场理由是上方偏离被拒绝，止损应放在价格重新接受 4245 上方并形成新成本区之后，而不是固定 5 点止损。

示例性路径五：同样的盘前结构中，价格突破 4230 后回撤浅，4250 上方形成新成交，Call Skew 抬升，卖方挂单撤出。此时主路径从“假突破观察”转为“上行接受/挤压”。交易员不应因为盘前预设 Call Wall 就强行逆向；执行规则要求路径跟随证据，而不是跟随早盘观点。

## 十七、仓位降级与止损逻辑

仓位不是对方向信心的奖励，而是对证据质量和执行条件的函数。方向判断再强，只要深度不足、价差扩大、路径未确认或事件风险未消化，仓位就必须降级。仓位分层可以写成四级。

| 仓位等级 | 条件 | 处理 |
|---|---|---|
| 0 级 | 只有位置，没有交易流验证 | 不交易 |
| 1 级 | 有初步被动流，但吸收/接受刚出现 | 小仓或观察 |
| 2 级 | 路径确认，回测成立，执行成本可控 | 标准仓位 |
| 降级状态 | 深度薄、价差大、事件未消化、滑点不可控 | 减半、等待或放弃 |

止损必须对应假设失效。回归型交易的失效，是偏离重新被接受；延续型交易的失效，是旧边界被收复并破坏新成本锚；衰竭型交易的失效，是最后一批强制流重新有效；中继型交易的失效，是新成本带被否定。若止损只按账户能忍受的金额设置，而不对应盘面失效，复盘就无法判断是路径错、执行错还是仓位错。

## 十八、复盘字段与执行检查

复盘不能只写盈亏。每一笔交易都应记录：盘前路径假设、主成本锚、CARCB 层级、期权环境、参与者假设、交易流验证、LOB 状态、入场触发、实际成交价、滑点、止损位置、路径是否失效、是否遵守降级规则、退出原因、最大顺向和最大逆向波动。

| 复盘字段 | 目的 |
|---|---|
| 路径假设 | 判断交易是否有明确研究对象 |
| 证据完整度 | 区分高质量亏损和低质量盈利 |
| 执行成本 | 评估滑点和深度问题 |
| 失效遵守 | 检查纪律是否稳定 |
| 规则违反 | 判断是否需要暂停交易或重写状态 |

示例性路径六：某次交易盈利，但复盘发现入场前没有明确 CARCB 层级，也没有 LOB 验证，只是价格随后顺向运行。这应被记为低质量盈利。另一笔交易亏损，但成本锚明确、路径验证完整、止损按失效执行，应被记为高质量亏损。长期改进依赖这种分类，而不是单笔盈亏情绪。

## 十九、盘后证伪禁止事项

盘后复盘不能把新叙事写回旧计划。若盘前主路径是“上方偏离失败回归”，盘中却发生“上方接受后中继”，复盘只能写原路径失效，并记录是否及时退出，不能把盈利或亏损重新解释成自己早已判断中继。盘后记录只能回答：行情是否符合原计划，何处失效，是否违反执行门禁。

执行层还要接受样本否定。若一类交易流或 LOB 验证长期不能改善 MAE、MFE、R multiple 或滑点稳定性，该验证变量必须降权；若某路径长期只能事后解释，不能在盘前定义触发和失效，必须从执行路径中删除。一个框架不能只要求市场被解释，也必须允许自己被样本否定。

## 二十、研究边界

执行规则能够做的，是提高证据门槛、限制仓位、定义失效和改善复盘；它不能消除突发消息、滑点、数据错误、流动性断层和模型失效。可观察但不能直接确认的部分，是某次入场是否正好对应真实强制流的来源。仍需后续样本检验的是：不同 checklist、仓位等级、路径类型与长期收益风险之间的关系。执行框架必须接受复盘结果修正：若某类路径长期无法覆盖滑点，不能因为理论合理就继续执行；若某个失效条件反复过宽或过窄，也必须通过样本调整。执行边界越清楚，复盘才越有统计意义，也越能避免事后自我辩护。

## 二十一、小结

执行章观察的核心变化，是研究假设能否转化为受约束的交易行为。它改变的是交易员自身的风险状态：从解释市场，转向等待关键区域、验证状态切换、控制仓位和记录失败。可能被迫行动的是市场参与者，但真正需要被约束的是交易员的解释冲动。盘前给观察地图，盘中给入场门禁，盘后给样本修正。只有位置、状态、执行证据、失效条件和执行成本同时成立，交易才进入权限范围；否则应降级、等待或放弃。
