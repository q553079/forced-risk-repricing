# 第十四篇：事件研究与样本复盘方法：如何把强制性风险再定价变成可验证框架

## 摘要

一个交易框架如果不能被记录、统计、证伪和迭代，就会退化为解释性语言。强制性风险再定价框架涉及成本锚、CARCB、保证金、期权 Gamma / Skew、波动率压缩、参与者约束、交易流、限价订单簿、收盘强度和跨市场确认。若这些变量只在事后被用来解释行情，框架会越来越像叙事；若它们能在盘前定义、盘中记录、盘后统计和周期性修正，框架才可能成为可验证研究方法。

本文建立事件研究（Event Study）与样本复盘（Sample Review）方法。目标不是证明框架永远正确，而是让每个假设都能被样本检验：某成本锚是否改善路径识别，某 CARCB 层级是否有统计优势，某交易流验证是否改善风险收益，某典型路径是否只能事后解释。最终形成“观察 → 假设 → 记录 → 统计 → 修正 → 执行降噪”的迭代循环。

## 一、为什么需要事件研究

单次复盘很容易事后归因。价格下跌后，交易员可以说是保证金压力；价格上涨后，可以说是负 Gamma 挤压；价格回归后，可以说是成本锚吸引。只要没有事前记录和样本统计，任何行情都能被解释，任何解释都很难被推翻。

事件研究的价值，是把解释变成可检验问题。

| 复盘方式 | 常见问题 | 事件研究修正 |
|---|---|---|
| 单次图形复盘 | 容易选择性归因 | 记录同类样本 |
| 事后找原因 | 变量随结果改变 | 盘前定义字段 |
| 只记成功案例 | 高估框架有效性 | 保留失败和无交易样本 |
| 只看盈亏 | 混淆过程和结果 | 区分证据质量与执行质量 |

样本统计可以检验 CARCB、成本锚、Gamma、Skew、LOB 验证是否真的改善交易结果。例如，如果深层 CARCB 触及后，加入 LOB 吸收确认的交易比不加入确认的交易具有更低 MAE 和更高 R multiple，那么 LOB 确认就有实证价值。若某成本锚评分很高但不能改善路径识别，它就应降权。

框架必须可证伪。不可证伪的交易语言，最终会成为心理安慰。可证伪的框架虽然会暴露错误，但也因此能够迭代。

## 二、事件研究样本字段

事件研究数据库应尽量结构化。以下字段不是为了复杂而复杂，而是为了让不同事件可以被比较。

| 字段 | 说明 | 类型 |
|---|---|---|
| date | 事件日期 | 日期 |
| instrument | 合约或标的 | 文本 |
| session | 隔夜、亚洲、欧洲、RTH、尾盘等 | 分类 |
| event type | CPI、FOMC、库存、无事件、技术偏离等 | 分类 |
| anchor type | VWAP、POC、Settlement、事件后成本区 | 分类 |
| anchor price | 成本锚价格 | 数值 |
| CARCB layer | 浅层、中层、深层、完全释放区 | 分类 |
| deviation unit | ATR、日内标准差、Expected Move 等 | 数值 |
| margin impact ratio | 偏离损益 / 保证金或缓冲估算 | 数值 |
| gamma regime | 正 Gamma、负 Gamma、不明 | 分类 |
| call wall / put wall | 相关期权集中区 | 数值/文本 |
| skew change | Put Skew / Call Skew 变化 | 分类/数值 |
| realized volatility | 实现波动率变化 | 数值 |
| implied volatility | 隐含波动率变化 | 数值 |
| book depth state | 深度稳定、下降、空洞、补单 | 分类 |
| aggressor imbalance | 主动成交净差 / CVD | 数值 |
| absorption / acceptance | 吸收、接受、拒绝、不明 | 分类 |
| close strength | 高位强收、低位弱收、收复、中性 | 分类 |
| intermarket confirmation | 确认、否定、冲突、不适用 | 分类 |
| path outcome | 路径结果分类 | 分类 |
| entry type | 回归、延续、衰竭、中继、无交易 | 分类 |
| R multiple | 以风险单位计的收益 | 数值 |
| MAE / MFE | 最大不利 / 最大有利波动 | 数值 |
| slippage | 滑点 | 数值 |
| mistake type | 提前、追价、忽略高周期等 | 分类 |
| notes | 简短备注 | 文本 |

字段不一定一开始就全部完美。若没有完整 LOB，可以记录降级字段；若没有实时 OI，可以盘后补充；若期权 Gamma 估计不确定，可以标记为“不明”。关键是口径一致，不能在成功案例里记录更多证据，在失败案例里少记录证据。

## 三、路径结果分类

路径结果必须标准化，否则样本无法统计。建议至少保留以下分类。

| 路径结果 | 定义 | 关键验证 |
|---|---|---|
| 偏离失败回归 | 价格离开成本锚后无法形成新成交，收复旧边界 | 推进效率下降、旧边界收复 |
| 风险约束释放延续 | 被动风险调整流被接受，价格进入下一层约束 | 新区域成交、回撤浅 |
| 深层衰竭反转 | 深层或完全释放区最后一轮强制流失效 | 净差背离、吸收、收复 |
| Gamma squeeze | 负 Gamma 与主动流同向放大 | 期权节点、撤单、回撤浅 |
| Put-side liquidation | Put Skew、下方 CARCB 和卖压共振 | 下方深度下降、卖压接受 |
| 新成本带中继 | 初次扩张后新区域二次压缩再扩张 | 新锚稳定、旧边界防守 |
| 流动性空洞 | 深度撤出导致少量成交大位移 | 价差扩大、深度恢复后验证 |
| 无效信号 | 条件不足或未形成状态切换 | 缺少成交/流动性确认 |

每个样本只能有一个主路径，但可以保留次级路径。例如，某事件可能同时有 Put-side liquidation 和流动性空洞，主路径取决于后续是否形成新成本区。如果深度恢复后价格仍被接受，主路径更接近 liquidation；若深度恢复后收复空洞区域，主路径应归为空洞失败。

## 四、如何验证 CARCB

CARCB 的验证不能只看触及后是否“反应”。应按层级和环境统计。

| 验证问题 | 样本统计方法 |
|---|---|
| 浅层、中层、深层的路径概率是否不同 | 按 CARCB layer 分组统计 path outcome |
| 不同品种阈值是否不同 | instrument 分组比较 |
| 不同时段表现是否不同 | session 分组比较 |
| 不同 Gamma 环境下是否不同 | gamma regime × CARCB layer 交叉 |
| 收盘强度是否影响隔夜延续 | close strength 与次日路径比较 |
| LOB 验证是否提高质量 | 有/无 absorption-acceptance 确认对比 |

一个有效的 CARCB 层级，应该在样本中体现差异。如果浅层、中层和深层触及后的路径概率没有差异，说明层级划分可能只是视觉距离，而不是风险距离。如果深层 CARCB 只有在负 Gamma 或低深度状态下才有延续优势，那么阈值模型必须纳入期权和 LOB 条件。

CARCB 验证还要记录无交易样本。触及深层但没有交易流，触及中层但深度稳定，触及浅层但快速回归，这些都是重要样本。只记录进入交易的案例，会让 CARCB 看起来比实际更有效。

## 五、如何验证执行

执行验证的核心问题是：框架是否改善了交易质量，而不只是改善了解释能力。每笔交易都应记录入场前证据是否齐全。

| 执行错误 | 定义 | 复盘修正 |
|---|---|---|
| 提前交易 | 吸收/接受尚未分化就入场 | 提高验证门槛 |
| 追价 | 在流动性最薄处入场 | 等回测或降级仓位 |
| 忽略高周期 | 低周期信号与高周期冲突仍放大仓位 | 加入周期门禁 |
| 忽略跨市场 | 单品种信号无外部确认仍按系统性处理 | 仓位降级 |
| 止损放错 | 止损不对应路径失效 | 重新定义失效条件 |
| 过度解释盘口 | 孤立成交被当成状态切换 | 回到成本锚和 CARCB |

执行统计至少应比较三个维度。第一，checklist 得分与 R multiple 的关系。第二，是否等待验证与 MAE 的关系。第三，滑点和 LOB 状态的关系。若高分交易长期表现更好，说明执行门槛有价值；若高分交易仍无法覆盖滑点，说明路径可能合理但不可交易。

## 六、可证伪性原则

可证伪性（Falsifiability）是本框架的底线。

| 假设 | 可证伪标准 | 处理 |
|---|---|---|
| 某成本锚有效 | 使用该锚不能改善路径识别 | 降权或废弃 |
| 某 CARCB 层级有效 | 层级之间路径概率无差异 | 重估阈值 |
| 某交易流验证有效 | 加入验证后风险收益无改善 | 剔除或降级 |
| 某典型路径有效 | 只能事后解释，不能盘前识别 | 不作为执行条件 |
| 某收盘强度有效 | 对隔夜延续无统计差异 | 降权 |

如果某成本锚不能改善路径识别，降权。若某 CARCB 层级没有统计优势，重估。若某交易流验证不能改善风险收益，剔除。若某路径只在事后能解释，不能盘前识别，不作为执行条件。

可证伪性不是削弱框架，而是保护框架。不能被推翻的框架，最终会吞噬所有结果；允许被推翻的框架，才有机会从样本中变得更精确。

## 七、事件研究流程

事件研究可以按以下流程执行：

`观察 → 假设 → 记录 → 统计 → 修正 → 执行降噪`

观察阶段，记录价格、成本锚、CARCB、期权、波动率、交易流和 LOB。假设阶段，写清主路径和失效条件。记录阶段，按字段填写，不因结果好坏改变口径。统计阶段，按品种、时段、事件、Gamma、收盘强度分组。修正阶段，降低无效变量权重，提高有效变量权重。执行降噪阶段，把统计结果变成盘前计划和盘中 checklist。

| 阶段 | 输出 | 常见错误 |
|---|---|---|
| 观察 | 原始变量 | 只看价格 |
| 假设 | 主路径和失效 | 多条路径混用 |
| 记录 | 结构化样本 | 成功案例记录更细 |
| 统计 | 分组结果 | 样本混杂 |
| 修正 | 阈值和权重变化 | 不愿承认无效 |
| 执行降噪 | 新 checklist | 复盘不进入执行 |

## 八、样本分组与最小样本集

样本必须分组。把所有品种、所有时段、所有事件混在一起，会得到无法执行的平均值。

建议至少按以下维度分组：

| 分组维度 | 示例 |
|---|---|
| 品种 | ES、NQ、CL、GC、ZN |
| 时段 | 隔夜、欧洲、RTH 开盘、午盘、尾盘 |
| 事件类型 | CPI、FOMC、库存、无事件 |
| CARCB 层级 | 浅层、中层、深层 |
| Gamma 环境 | 正 Gamma、负 Gamma、不明 |
| 收盘强度 | 高位强收、低位弱收、收复、中性 |
| 路径结果 | 回归、延续、衰竭、空洞、无效 |

最小样本集不应只包含交易。至少要包括：入场交易、观察未交易、路径成立但放弃、路径失败但避免亏损。这样才能评估过滤系统的机会成本和保护价值。

## 九、示例性路径：样本记录如何工作

示例性路径一：某 CPI 日，NQ 在数据前主成本锚为 18200。数据后上冲至 18320，进入上方中层 CARCB，接近 Call Wall。Call Skew 轻微抬升，但主动买入推进效率下降，上方 LOB 补单明显。价格 30 分钟后收复 18260，收盘回到 18210。样本记录为：event type = CPI，anchor type = event pre POC，CARCB layer = middle up，gamma regime = positive / unclear，absorption = yes，close strength = neutral-to-rejection，path outcome = 偏离失败回归。若交易员做回归，entry type = rejection，R multiple、MAE、MFE 和 slippage 进入统计。

示例性路径二：某 FOMC 日，ES 跌破 5200 成本锚，进入下方深层 CARCB。Put Skew 走陡，VIX 上行，US10Y 上行，book depth state = thin，aggressor imbalance = strong sell，close strength = low weak close。样本记录为风险约束释放延续或 Put-side liquidation，取决于后续是否形成新成本带。若次日低开后继续接受下方价格，该路径样本进入隔夜延续统计。

## 十、从数据库到执行规则

事件研究的最终目标，不是做一个漂亮表格，而是更新执行规则。若样本显示“深层 CARCB + 负 Gamma + 低位弱收”的隔夜延续概率较高，盘前计划应提高该组合的顺向权重。若样本显示“午盘突破 + 无新成本区”的失败概率高，盘中 checklist 应禁止追午盘第一段突破。

| 样本发现 | 执行规则变化 |
|---|---|
| 高评分成本锚改善路径识别 | 盘前必须标注主锚 |
| 无 LOB 验证交易 MAE 偏高 | 入场前必须等待吸收/接受 |
| 低流动性空洞滑点过大 | 空洞路径仓位降级 |
| 收盘强度影响次日延续 | 尾盘复盘进入次日计划 |
| 跨市场不确认样本表现差 | 单品种信号仓位降级 |

复盘只有进入下一次执行，才有价值。否则复盘只是事后评价，不能改善交易。

### 数据质量：哪些样本不能直接进入统计

事件研究最怕的不是样本少，而是样本质量不一致。若一部分样本有逐笔成交和 LOB 记录，另一部分只有 K 线截图；一部分样本事前写了成本锚，另一部分事后补锚；一部分样本记录真实滑点，另一部分按理想价格计算，统计结果会被污染。数据库必须先区分“可统计样本”和“仅可描述样本”。

| 样本问题 | 风险 | 处理方式 |
|---|---|---|
| 成本锚事后补写 | 容易选择最有利锚点 | 标记为低质量，不进入锚点有效性统计 |
| LOB 数据缺失 | 无法判断吸收/接受 | 可用于方向统计，不用于微观验证统计 |
| 事件窗口不一致 | 结果受观察期选择影响 | 统一窗口或单独分组 |
| 未记录滑点 | 高估执行质量 | 不进入执行收益统计 |
| 只记录交易样本 | 忽略无交易过滤价值 | 增加 no trade 样本 |
| 忽略失败路径 | 路径概率被高估 | 强制记录失效样本 |

研究者应为每条样本设置 quality flag，例如 A 类样本包含成本锚、CARCB、交易流、LOB、收盘和执行结果；B 类样本缺少 LOB 但有成交与收盘；C 类样本只有图形描述。不同质量样本可以服务不同问题，但不能混用。A 类可以研究执行规则，B 类可以研究路径方向，C 类最多作为案例线索。

### 统计读法：不要只看胜率

强制性风险再定价框架检验的重点，不应只有胜率。一个条件组合即使胜率不高，也可能通过降低 MAE、减少滑点暴露、提高 MFE 或改善盈亏分布左尾而具有价值。相反，一个看似高胜率的条件，如果主要发生在极窄波动环境，收益空间不足，或一旦失败就出现巨大滑点，也不适合作为核心执行条件。

| 指标 | 解释 | 适合检验的问题 |
|---|---|---|
| Path hit rate | 路径分类是否兑现 | CARCB 层级是否有区分度 |
| MAE | 入场后最大不利波动 | 验证等待是否降低错误暴露 |
| MFE | 入场后最大有利波动 | 路径是否提供足够空间 |
| R multiple | 按初始风险标准化的结果 | 执行规则是否有效 |
| Slippage | 理论价与成交价差异 | 流动性空洞是否应降级 |
| Time to validation | 从触发到验证所需时间 | 等待规则是否过慢或过快 |
| No trade value | 被过滤样本后续表现 | 过滤条件是否有价值 |

尤其需要记录 no trade 样本。很多框架的真实价值不在于增加入场，而在于避免低质量入场。若某类“跨市场不确认 + 无新成本锚 + 午盘突破”的样本大多在后续回归，那么不交易本身就是有效结果。事件研究如果只统计已经下单的样本，会低估过滤系统的贡献。

### 示例性路径三：可证伪的样本迭代

示例性路径：研究者最初假设“深层 CARCB + 低位弱收”会提高次日下行延续概率。连续记录 60 个样本后发现，只有在负 Gamma 或 Put Skew 走陡环境下，该组合的延续质量较高；在正 Gamma 环境下，低位弱收次日反而更容易出现均值回归。此时正确处理不是把失败样本解释掉，而是把原假设拆分为两个条件组合：深层 CARCB + 低位弱收 + 负 Gamma，和深层 CARCB + 低位弱收 + 正 Gamma。

这条示例说明，框架迭代不是不断增加变量，而是让变量承担可检验的分组作用。若加入 Gamma 后，组间结果仍无差异，就说明该变量在该品种和周期上没有提供有效信息；若差异显著，再进入执行规则测试。研究者不能因为一个变量“理论上重要”就让它永久保留在 checklist 中。

### 从证伪到规则降级

样本统计的结论不应只有“保留”或“删除”。在交易实务中，更常见的是规则降级。某个条件可能不足以支持隔夜持有，但足以支持日内短线；某个路径可能不适合标准仓位，但可以作为观察下一层 CARCB 的预警；某个期权节点可能不能预测方向，但能提示滑点和波动率风险。

| 统计结论 | 规则处理 |
|---|---|
| 条件显著改善路径识别和执行质量 | 保留为核心门槛 |
| 条件改善方向但滑点较大 | 保留观察，执行仓位降级 |
| 条件只在特定时段有效 | 加入时段限制 |
| 条件只在特定 Gamma / Skew 环境有效 | 加入环境过滤 |
| 条件无稳定差异 | 从执行条件降为备注 |
| 条件导致过度交易或 MAE 扩大 | 删除或反向作为风险提示 |

这样的规则降级机制，可以避免研究结论过于二元。市场结构不是实验室里的封闭系统，同一个变量在不同品种、时段、流动性和制度环境下可能有不同权重。事件研究的任务，是让这些权重逐渐从经验判断变成可讨论、可修改、可复盘的规则。

## 十一、研究边界

有制度或公开研究支持的部分，是市场存在保证金、结算、期权对冲、波动率、订单流和流动性冲击等机制；事件研究方法也可以用于检验事件前后变量变化。可观察但不能直接确认身份的部分，是每一段成交背后的具体账户、dealer 净仓位、基金 VaR 使用率和 HFT 撤单动机。需要事件研究验证的交易假设，是 CARCB 层级、成本锚评分、Gamma / Skew 共振、LOB 验证、收盘强度和跨市场确认是否真正改善风险收益。数据库不能消除不确定性，但能减少自我解释。

## 十二、小结

强制性风险再定价要成为研究框架，必须离开单次复盘和事后叙事。每个概念都应被记录，每个路径都应被分类，每个执行错误都应进入统计，每个阈值都应允许被修正。

最终目标不是证明自己对，而是建立可迭代研究循环：观察、假设、记录、统计、修正、执行降噪。只有这样，框架才不会变成解释所有行情的语言，而能成为筛选交易、降低噪音、修正错误的实务工具。
